# ליגת ראש העיר 2026 — אתר הליגה

אתר להצגת לוח המשחקים, התוצאות וטבלאות הבתים של ליגת ראש העיר — מחלקת הספורט קריית ביאליק.
אתר סטטי + פונקציית שרת אחת (`api/results.js`) + Upstash Redis (מפתח אחד) — מיועד לפריסה ב-Vercel.

## עמודים
- `index.html` — לוח משחקים לפי כיתה (תאריך, שעה, מגרש, תוצאה) + טבלאות בתים מחושבות מהתוצאות, כולל מסנן "משחקים חיים" (שעת התחלה, תוצאה חיה וכובשים) ורענון אוטומטי.
- `scorers.html` — מלך השערים: טבלת כובשים גלובלית או לפי כיתה (גול עצמי לא נספר).
- `admin.html` — עמוד ניהול (משתמש: `admin`, סיסמה: `admin`): התחלת משחק ← הוספת שערים עם בחירת כובש (או "גול עצמי") ← סיום משחק. כל פעולה נשמרת אוטומטית.

## מצבי משחק
לכל משחק אחד משלושה מצבים: **טרם שוחק** (אין רשומה) | **חי** (`st:"L"` — תוצאה מתעדכנת + כובשים) | **הסתיים** (`st:"E"` — תוצאה סופית).
מבנה תוצאה: `{ st, h, a, goals: [{side:"h"|"a", player:"שם"|null}], startedAt }` — `player:null` = גול עצמי.
תוצאות ישנות בפורמט `{h,a}` בלבד נחשבות משחק שהסתיים. רק משחקים שהסתיימו נספרים בטבלאות הבתים.

## רשימת השחקנים
`js/players.js` נבנה מנתוני האמת שב-`../teams.json` (92 קבוצות, כולל מיפוי הבדלי איות מול `data.js`).
עדכון: עורכים את `teams.json` ומריצים `node ../gen-players.mjs --write` (בלי `--write` — דוח התאמות בלבד),
או עורכים ישירות: `PLAYERS[כיתה][שם קבוצה] = ["שם מלא", ...]` (שם הקבוצה חייב להתאים בדיוק ל-`js/data.js`).
שתי קבוצות עדיין ללא סגל ב-teams.json: אריות הגבעה (ו') ומכבי רכבות FC (ח').

## איך עובדות התוצאות
- **ב-Vercel (מומלץ):** `GET /api/results` קורא את התוצאות מ-Redis; שמירה בעמוד הניהול שולחת `POST /api/results` (עם סיסמת המנהל, הנבדקת בשרת) — והתוצאות מתעדכנות מיידית לכל הגולשים.
- **ללא שרת (מצב סטטי):** האתר נופל לאחור לקריאה מ-`data/results.json` + שמירה מקומית בדפדפן בלבד.

## פריסה ל-Vercel
```bash
cd site
vercel integration add upstash   # יצירת Redis (חינמי) + הזרקת משתני סביבה
vercel deploy --prod
```
אופציונלי: קביעת סיסמת מנהל אמיתית עם משתנה סביבה `ADMIN_PASSWORD` (ברירת מחדל: `admin`).
שימו לב: הסיסמה בעמוד ההתחברות נבדקת גם בצד השרת — עדכון תוצאות בלי הסיסמה הנכונה יידחה.
